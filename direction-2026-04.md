# MCH Studio — Direction

MCH Studio is a workflow-first creative production system.

The current priority is not a better app. The priority is a reliable operating loop that turns good ideas into published assets.

## Principle

Results matter more than infrastructure.

The app, Supabase, Remotion, and automation are only useful if they help a production run move toward a published, scheduled, or intentionally killed asset.

## Active Source of Truth

`studio-runs/`

## Current Test

Run three real MCH ideas through the local folder workflow:

1. Capture idea
2. Promote to production run
3. Draft assets
4. Prepare publish kit
5. Publish, schedule, or kill
6. Log what happened

If this makes execution easier, consider rebuilding `/studio` as a cockpit over those files.

## Out of Scope

- Supabase persistence
- Hosted app as source of truth
- Autopublishing
- Elaborate dashboards
- Remotion rendering before the script/payload workflow earns it
